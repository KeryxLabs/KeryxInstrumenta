# Contributing to KeryxInstrumenta

Thanks for helping improve KeryxInstrumenta.

## Quick start

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make focused changes with tests.
4. Run local validation:

```bash
dotnet test ./src/sttp-mcp.Tests/sttp-mcp.Tests.csproj
```

5. Open a pull request with a clear title and summary.

## Contribution guidelines

- Keep changes scoped to one concern per PR.
- Follow existing code style and naming.
- Add or update tests for behavior changes.
- Update docs (`README`, `ARCHITECTURE`, or protocol docs) when behavior changes.

## Pull request checklist

- [ ] Builds successfully
- [ ] Tests pass
- [ ] Docs updated (if needed)
- [ ] No unrelated refactors

## Reporting issues

Use GitHub Issues for bugs and feature requests.
For security disclosures, see `SECURITY.md`.
